createConfig

The createConfig function is used to create a Rsbuild configuration object setup for Angular applications.

It takes an optional RsbuildConfig object as an argument, which allows for customization of the Rsbuild configuration.

Examples

PluginAngularOptions

The PluginAngularOptions object is an object that contains the following properties: 

  
export interface PluginAngularOptions {
    root: string;
    index: string;
    browser: string;
    server?: string;
    ssrEntry?: string;
    polyfills: string[];
    assets: string[];
    styles: string[];
    scripts: string[];
    jit: boolean;
    inlineStylesExtension: 'css' | 'scss' | 'sass';
    tsconfigPath: string;
    hasServer: boolean;
    useParallelCompilation: boolean;
    useHoistedJavascriptProcessing: boolean;
}

root

Type: string

The root directory of the project. This is the directory where the package.json file is located.

index

Type: string

The path to the index file. This is used to determine the index file to use.

browser

Type: string

The path to the browser file. This is used to determine the browser file to use.

server

Type: string

The path to the server file. This is used to determine the server file to use.

ssrEntry

Type: string

The path to the server entry file. This is used to determine the server entry file to use.

polyfills

Type: Array<string>

An array of polyfills to include in the build. This is used to determine the polyfills to include in the build.

assets

Type: Array<string>

An array of assets to include in the build. This is used to determine the assets to include in the build.

styles

Type: Array<string>

An array of styles to include in the build. This is used to determine the styles to include in the build.

scripts

Type: Array<string>

An array of scripts to include in the build. This is used to determine the scripts to include in the build.

jit

Type: boolean

A boolean value indicating whether to use the JIT mode. This is used to determine whether to use the JIT mode.

inlineStylesExtension

Type: 'css' | 'scss' | 'sass'

The inline styles extension to use. This is used to determine the inline styles extension to use.

tsconfigPath

Type: string

The path to the TypeScript configuration file. This is used to determine the TypeScript configuration file to use.

hasServer

Type: boolean

A boolean value indicating whether the project has a server. This is used to determine whether the project has a server.

useParallelCompilation

Type: boolean

A boolean value indicating whether to use parallel compilation. This is used to determine whether to use parallel compilation.

useHoistedJavascriptProcessing

Type: boolean

A boolean value indicating whether to use hoisted javascript processing. This is used to determine whether to use hoisted javascript processing.
